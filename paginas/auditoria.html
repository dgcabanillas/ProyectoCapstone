Auditoria de instalación

  • Tuvimos problemas al unir distintos proyectos, podriamos hacerlo a traves de un servior, pero preferimos usar las nubes de gns3, el inconveniente es que cada uno tenía que tener todo el projecto en su computadoras, para ello usamos un controlador de versiones como github.
  • Claramente hay una gran cantidad de rutas, esto trae problemas tales como si se crea más facultades o mas usuarios, podriamos tener que reconfigurar toda la red, para ellos cada facultad cuenta con su propio dhcp.
  • Por un motivo de seguridad, y de comodidad, se usaron switch para crear vlan(s), y también para tener un mejor tráfico entre usuarios de una misma facultad, esto también incluye el uso individual de una PAS.
  • Conectamos los campus por redes wlan, estan pueden estar muy alejadas, sin embargo tuvimos problemas sobre la configuración en GNS3, encontramos una solución por frame relay switching que no se ha implementado en esta entrega.
